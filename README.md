# Bank Cards Management System (MVP)

Backend-приложение для управления банковскими картами, счетами и пользователями.  
Реализована аутентификация через JWT, ролевая модель (ADMIN/USER), шифрование номеров карт (AES‑GCM), переводы между своими картами, поиск и пагинация.

---

## Основные возможности
- Регистрация и аутентификация пользователей (JWT).
- Администратор:
    - создание, блокировка, активация, удаление карт;
    - управление пользователями (CRUD, блокировка/активация);
    - просмотр всех карт с пагинацией.
- Пользователь:
    - просмотр своих карт (поиск по последним 4 цифрам + пагинация);
    - запрос на блокировку карты;
    - переводы между своими картами;
    - просмотр баланса карты.
- Шифрование номера карты (AES‑GCM) и маскировка в ответах (`**** **** **** 1234`).
- Поиск карт по хешу последних 4 цифр (без расшифровки).
- Валидация входящих данных и глобальная обработка ошибок.
- Документация API через Swagger/OpenAPI.

## Переменные окружения (.env)

Все конфиденциальные данные (пароли, ключи шифрования, JWT-секреты) вынесены в файл `.env`.  
**Этот файл не должен попадать в Git** — добавьте его в `.gitignore`.

### Создание `.env`
В корне проекта (рядом с `pom.xml`) создайте файл `.env` со следующим содержимым:

```env
# База данных (локальная)
DB_URL=
DB_USERNAME=
DB_PASSWORD=
DB_NAME=
DB_PORT=

# Ключи шифрования и JWT
APP_CRYPTO_KEY=
JWT_SECRET=
JWT_EXPIRATION=

# HMAC секрет для хеширования (для поиска по последним 4 цифрам)
HMAC_SECRET=

# Данные для администратора (создаётся при первом запуске)
ADMIN_EMAIL=
ADMIN_PASSWORD=
```

> **Важно:**
> - Не используйте пробелы вокруг `=`.
> - Значения не должны быть в кавычках (кроме случаев с пробелами внутри значения).
> - Убедитесь, что файл `.env` добавлен в `.gitignore`.

# Запуск приложения

## Локальный запуск (без Docker)

### через терминал (экспорт вручную)
```bash
export DB_URL=jdbc:postgresql://localhost:5432/bank
export DB_USERNAME=postgres
export DB_PASSWORD=ваш_пароль
export APP_CRYPTO_KEY=...
export JWT_SECRET=...
mvn spring-boot:run
```

#### через IntelliJ IDEA
1. Откройте **Run → Edit Configurations**.
2. В вашей конфигурации Spring Boot найдите поле **Environment variables**.
3. Вставьте переменные в формате:
   ```
   DB_URL=jdbc:postgresql://localhost:5432/bank;DB_USERNAME=postgres;DB_PASSWORD=ваш_пароль;APP_CRYPTO_KEY=...;JWT_SECRET=...;JWT_EXPIRATION=86400000
   ```

---

### Запуск через Docker Compose

1. Убедитесь, что файл `.env` лежит рядом с `docker-compose.yml`.
2. Соберите JAR (если не собирали):
   ```bash
   mvn clean package -DskipTests
   ```
3. Запустите контейнеры:
   ```bash
   docker compose up --build
   ```
   > **Важно:** флаг `--build` **всегда пересобирает образ** перед запуском. Это обязательно, если вы изменили код, JAR, `Dockerfile` или зависимости.  
   > Если код не менялся, можно использовать `docker compose up` (без `--build`) — он запустит уже существующий образ быстрее, но не подхватит изменения.
4. Приложение будет доступно по адресу: `http://localhost:8080`.  
   После запуска документация API доступна по ссылке: [Swagger UI](http://localhost:8080/swagger-ui/index.html)

**Остановка контейнеров:**
```bash
docker compose down
```

**Остановка и удаление данных БД (volume):**
```bash
docker compose down -v
```

## Безопасность
- Пароли пользователей хранятся в виде BCrypt‑хеша.
- Номера карт шифруются алгоритмом AES‑GCM (256 бит) с уникальным IV для каждой записи.
- Поиск карт по последним 4 цифрам осуществляется через HMAC‑хеш, что позволяет искать без расшифровки номеров.
- JWT‑токены подписываются секретным ключом, указанным в переменной `JWT_SECRET`.

## Возможные проблемы и решения

| Проблема | Решение |
|----------|---------|
| Ошибка подключения к БД | Проверьте, что PostgreSQL запущен и параметры в `.env` верны. |
| Шифрование номера карты не работает | Убедитесь, что `APP_CRYPTO_KEY` — корректный 256‑битный ключ в Base64 (ровно 32 байта). |
| JWT не валиден | Проверьте время жизни токена и корректность заголовка `Authorization`. |
| Docker не видит переменные из `.env` | Убедитесь, что `.env` лежит рядом с `docker-compose.yml` и содержит нужные переменные. |
| Локальный запуск не подхватывает `.env` | Используйте скрипт для экспорта переменных перед запуском. |
| Liquibase не применяется | Включите `spring.liquibase.enabled=true` и `spring.jpa.hibernate.ddl-auto=none` в `application.yml`. |
